# El Club de la Lucha - Agent Instructions

## Project Overview
Web app for a Pokémon VGC competitive gaming club. Single-page application with vanilla JS, no build tools, no frameworks.

## Tech Stack
- **HTML5** - Single `index.html` file
- **CSS3** - `styles.css` with CSS custom properties (dark theme)
- **JavaScript** - Vanilla JS, no modules, no bundler
- **Storage** - localStorage + Supabase (PostgreSQL)

## File Structure
```
├── index.html      # Single page, all modals and sections inline
├── app.js          # All JS logic
├── styles.css      # All styles
├── data.json       # Legacy data file (no longer used)
├── logoClub.png    # Club logo
└── AGENTS.md       # This file
```

## Architecture

### Main Objects (app.js)
- **`DB`** - Database layer. Handles localStorage cache + Supabase sync.
  - `DB._supabase` = Supabase client instance
  - `DB._cache` = `{ sesiones: [], deberes: [] }`
  - `DB.cargar()` - Load from Supabase or fallback to localStorage
  - `DB.set(clave, datos)` - Save to cache + localStorage + sync to Supabase
  - `DB._syncSupabase(clave, datos)` - Delete all + reinsert into Supabase table
  - `DB._cargarDesdeSupabase()` - Load both tables from Supabase
  - `DB._getConfig()` - Read Supabase config from localStorage
  - `DB._ensureConfig()` - Set default Supabase config if none exists
- **`Config`** - Supabase config modal (URL + anon key)
- **`Sesiones`** - Sessions CRUD (theorica/practica). Filtered view, cards grid.
- **`Deberes`** - Homework/tasks CRUD. Toggle completed, priority, deadlines.
- **`POKEDEX`** - Map of Pokémon names → sprite info. Values are either numeric IDs (national dex) or string paths (for forms with gender/variant sprites). VGC competitive only.

### Data Model
```javascript
// Sesión (table: sesiones)
{
  id: string,        // Date.now base36 + random
  fecha: string,     // "YYYY-MM-DD"
  tipo: "teorica" | "practica",
  tema: string,
  descripcion: string,
  youtube: string,   // URL
  pokepastes: [{ url, raw, equipo }],  // JSONB
  asistentes: string
  // Note: fechaLimite is mapped to fecha_limite in Supabase
}

// Deber (table: deberes)
{
  id: string,
  titulo: string,
  descripcion: string,
  pokepastes: [{ url, raw, equipo }],  // JSONB
  fechaLimite: string,  // "YYYY-MM-DD" → mapped to fecha_limite
  prioridad: "normal" | "alta",
  completado: boolean
  // Note: created_at is auto-generated by Supabase
}
```

### Key Functions
- `parsearPoképaste(texto)` - Parses pokepaste text, extracts Pokémon names, handles gender/form markers (F/M/♀/♂) to look up correct sprite in POKEDEX. Returns `{nombre, sprite}` objects where sprite is either a numeric ID or string path.
- `normalizarNombre(str)` - Normalizes names (lowercase, remove accents/symbols, guiones→espacios)
- `renderEquipo(equipo, urlPaste)` - Renders sprite images. Supports both `{sprite}` (new) and `{id}` (legacy) formats via `spriteUrl()`.
- `spriteUrl(sprite)` - Resolves sprite to full PokeAPI URL. If numeric → `{BASE}{id}.png`. If string → `{BASE}{path}`.
- `renderYouTube(url)` - Renders YouTube link (not embedded)
- `escHtml(str)` - XSS-safe HTML escaping
- `mostrarEstado(tipo, mensaje)` - Status bar notifications (ok/error/loading)

### Supabase Sync
- Uses Supabase JS v2 CDN (`https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2`)
- Supabase client: `supabase.createClient(url, anonKey)`
- Config stored in localStorage (`club_config`) with `{ url, anonKey }`
- Default config auto-set via `DB._ensureConfig()`
- Sync strategy: delete all rows from table, reinsert local data
- Tables: `sesiones`, `deberes` with RLS enabled (allow all for anon)

### Field Mapping (JS ↔ Supabase)
- `fechaLimite` (JS) → `fecha_limite` (Supabase)
- `created_at` (Supabase) → auto-generated, stripped on insert

## Conventions
- **No imports/exports** - Everything is global, loaded via single `<script>` tag
- **No build step** - Edit files directly, open index.html in browser
- **Spanish** - UI text and code comments in Spanish
- **No frameworks** - Pure vanilla JS, no React/Vue/etc.
- **CSS variables** for theming (`--accent`, `--bg-card`, etc.)
- **Event handlers** inline in HTML (`onclick="Sesiones.abrirModal()"`)
- **Async DB operations** - All `DB.set()` calls must be awaited

## How to Run
- Open `index.html` directly in browser, or
- Use a local server (e.g., VS Code Live Server)

## Supabase Setup
Default config is hardcoded in `DB._ensureConfig()`. User can override via ⚙ button:
1. Supabase project URL (https://xxxxx.supabase.co)
2. Anon key (public, safe for client-side)

SQL tables created:
- `sesiones` with RLS policies allowing all for anon
- `deberes` with RLS policies allowing all for anon

## Notes
- Pokémon sprites from: `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png` for numeric IDs, or `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{path}` for string paths (e.g., `other/home/10248.png`)
- Poképastes URL: `https://pokepast.es/`
- The POKEDEX object covers Gen 1-9 competitive VGC Pokémon
- Gender/form variants use string paths in POKEDEX (e.g., `"basculegion (f)": "other/home/10248.png"`)
- Modals are shown/hidden via `display: none/flex`
- Sections switched via `.section.active` CSS class
- No tests, no linting configured
